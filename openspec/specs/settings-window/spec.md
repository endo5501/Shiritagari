# Capability: settings-window

## Purpose

設定ウィンドウ機能。トレイメニューから設定画面を開き、LLMプロバイダ・監視設定・外観設定などをGUIで変更・保存できる。

## Requirements

### Requirement: トレイメニューから設定ウィンドウを開く
システムはトレイメニューに「Settings」項目を表示しなければならない（SHALL）。クリック時にメインウィンドウとは別の設定ウィンドウを開かなければならない（SHALL）。設定ウィンドウが既に開いている場合は、新規ウィンドウを作成せずに既存ウィンドウにフォーカスしなければならない（SHALL）。

#### Scenario: 設定ウィンドウを開く
- **WHEN** ユーザがトレイメニューの「Settings」をクリックした時
- **THEN** 設定ウィンドウ（約500x600）が表示され、現在の設定値が読み込まれている

#### Scenario: 設定ウィンドウが既に開いている場合
- **WHEN** 設定ウィンドウが既に開いている状態で「Settings」をクリックした時
- **THEN** 既存の設定ウィンドウにフォーカスが移る

### Requirement: 現在の設定を読み込んで表示する
システムは設定ウィンドウを開いた際に、現在のランタイム設定を読み込んで各フォームフィールドに反映しなければならない（SHALL）。

#### Scenario: 設定の読み込み
- **WHEN** 設定ウィンドウが開かれた時
- **THEN** 現在の AppConfig の値がフォームの初期値として表示される

### Requirement: LLMプロバイダの設定
システムはLLMプロバイダ（Ollama / Claude / OpenAI）をドロップダウンで選択可能にしなければならない（SHALL）。選択したプロバイダに応じて、必要な入力フィールドを動的に切り替えなければならない（SHALL）。

#### Scenario: Ollamaプロバイダ選択時のフォーム
- **WHEN** プロバイダとして「Ollama」を選択した時
- **THEN** Base URL入力フィールドとモデル選択ドロップダウンが表示される

#### Scenario: Claudeプロバイダ選択時のフォーム
- **WHEN** プロバイダとして「Claude」を選択した時
- **THEN** APIキー環境変数名の入力フィールドとモデル入力フィールドが表示される

#### Scenario: OpenAIプロバイダ選択時のフォーム
- **WHEN** プロバイダとして「OpenAI」を選択した時
- **THEN** Base URL入力フィールド、APIキー環境変数名の入力フィールド、モデル入力フィールドが表示される

### Requirement: Ollamaモデル一覧からの選択
システムはOllamaプロバイダ選択時に、Ollama APIからローカルにインストール済みのモデル一覧を取得し、ドロップダウンで選択可能にしなければならない（SHALL）。モデル一覧の取得に失敗した場合は、手動入力へのフォールバックを提供しなければならない（SHALL）。

#### Scenario: モデル一覧の取得と表示
- **WHEN** Ollamaプロバイダが選択されている状態で設定画面が表示された時
- **THEN** Ollama API (`/api/tags`) からモデル一覧を取得し、ドロップダウンに表示する

#### Scenario: モデル一覧の更新
- **WHEN** ユーザが「モデル一覧を更新」ボタンをクリックした時
- **THEN** Ollama API から最新のモデル一覧を再取得し、ドロップダウンを更新する

#### Scenario: Ollama未起動時のフォールバック
- **WHEN** Ollama APIへの接続に失敗した時
- **THEN** エラーメッセージを表示し、モデル名の手動入力フィールドにフォールバックする

#### Scenario: Base URL変更時のモデル一覧再取得
- **WHEN** Ollama Base URLが変更された時
- **THEN** 新しいBase URLに対してモデル一覧を再取得する

### Requirement: 監視設定の変更
システムはポーリング間隔（分）とアプリの許可リスト・拒否リストを設定画面から変更可能にしなければならない（SHALL）。

#### Scenario: ポーリング間隔の変更
- **WHEN** ポーリング間隔を変更して保存した時
- **THEN** 次のポーリングサイクルから新しい間隔が適用される

#### Scenario: アプリリストの編集
- **WHEN** 許可リストまたは拒否リストにアプリ名を追加・削除して保存した時
- **THEN** 次のポーリングサイクルから新しいフィルタ設定が適用される

### Requirement: 外観設定の変更
システムはマスコット画像のパスを設定画面から変更可能にしなければならない（SHALL）。

#### Scenario: マスコット画像パスの設定
- **WHEN** マスコット画像パスを入力して保存した時
- **THEN** メインウィンドウのマスコット表示が更新される

### Requirement: 設定の保存と即時反映
システムは設定保存時に config.toml へ永続化し、同時にランタイムの AppState を更新しなければならない（SHALL）。設定画面で管理しない項目（inference/chat個別設定、confidence、redaction_patterns等）は既存値を保持しなければならない（SHALL）。

#### Scenario: 設定の保存
- **WHEN** ユーザが「保存」ボタンをクリックした時
- **THEN** 設定が config.toml に書き込まれ、ランタイム設定が更新され、設定ウィンドウが閉じる

#### Scenario: 未管理項目の保持
- **WHEN** config.toml に inference_provider 等の上級者向け設定が手動で記述されている状態で設定を保存した時
- **THEN** 設定画面で管理しない項目は元の値のまま保持される

#### Scenario: 設定のキャンセル
- **WHEN** ユーザが「キャンセル」ボタンをクリックまたはウィンドウを閉じた時
- **THEN** 変更は破棄され、設定は変更されない
