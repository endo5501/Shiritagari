## Purpose

学習した知識（パターン記憶・エピソード記憶・ユーザプロフィール）をブラウザで閲覧できる Web UI を提供する。

## Requirements

### Requirement: 組み込みHTTPサーバの起動
システムはアプリ起動時に localhost 限定で HTTP サーバを起動しなければならない（SHALL）。ポート 14789 を起点に空きポートを自動探索する。

#### Scenario: デフォルトポートでの起動
- **WHEN** アプリが起動し、ポート 14789 が空いている時
- **THEN** localhost:14789 で HTTP サーバを起動する

#### Scenario: ポート衝突時のフォールバック
- **WHEN** ポート 14789 が使用中の時
- **THEN** 14790 から最大 100 ポート分を探索し、最初に空いているポートで起動する

### Requirement: トレイメニューからブラウザを開く
システムトレイメニューに「Knowledge Base」項目を追加し、クリック時にデフォルトブラウザで Web UI を開かなければならない（SHALL）。

#### Scenario: Knowledge Base メニュークリック
- **WHEN** ユーザがトレイメニューの「Knowledge Base」をクリックした時
- **THEN** デフォルトブラウザで `http://localhost:{実際のポート}/` を開く

### Requirement: Patterns 一覧ページ
Web UI のルート (`/`) でアクティブなパターン記憶の一覧を表示しなければならない（SHALL）。

#### Scenario: パターン一覧の表示
- **WHEN** ユーザが `/` にアクセスした時
- **THEN** アクティブなパターンの一覧（アプリ名、タイトル条件、意味、確信度、最終確認日）をページング付きで表示する

#### Scenario: パターンが0件の場合
- **WHEN** アクティブなパターンが存在しない時
- **THEN** 「まだパターンが学習されていません」というメッセージを表示する

### Requirement: Episodes 一覧ページ
Web UI の `/episodes` でエピソード記憶の一覧を新しい順に表示しなければならない（SHALL）。

#### Scenario: エピソード一覧の表示
- **WHEN** ユーザが `/episodes` にアクセスした時
- **THEN** エピソードの一覧（日時、アプリ名、タイトル、時間、質問、回答、タグ）を新しい順にページング付きで表示する

#### Scenario: エピソードが0件の場合
- **WHEN** エピソードが存在しない時
- **THEN** 「まだエピソードが記録されていません」というメッセージを表示する

### Requirement: Profile ページ
Web UI の `/profile` でユーザプロフィールを表示しなければならない（SHALL）。

#### Scenario: プロフィールの表示
- **WHEN** ユーザが `/profile` にアクセスした時
- **THEN** ユーザプロフィール（職業、スキル、関心事、メモ、最終更新日）を表示する

#### Scenario: プロフィールが未設定の場合
- **WHEN** ユーザプロフィールがまだ保存されていない時
- **THEN** 「プロフィールはまだ設定されていません」というメッセージを表示する

### Requirement: ページング
Patterns と Episodes の一覧ページは1ページあたり 20 件表示し、ページナビゲーションを提供しなければならない（SHALL）。

#### Scenario: ページ切り替え
- **WHEN** ユーザがページナビゲーションで別ページを選択した時
- **THEN** 該当ページのデータを表示し、現在のページ番号と総ページ数を表示する

#### Scenario: 1ページ以内のデータ量
- **WHEN** データが 20 件以下の時
- **THEN** ページナビゲーションを表示しない
