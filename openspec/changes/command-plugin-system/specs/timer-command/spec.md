## ADDED Requirements

### Requirement: タイマーの設定
システムは `/timer <時間>` コマンドで指定された時間のワンショットタイマーを設定しなければならない（SHALL）。

#### Scenario: 時間指定でタイマー設定
- **WHEN** ユーザが `/timer 3時間` と入力した時
- **THEN** 3時間のタイマーを開始し、「タイマーを3時間に設定したよ！」と応答する

#### Scenario: 分指定でタイマー設定
- **WHEN** ユーザが `/timer 30分` と入力した時
- **THEN** 30分のタイマーを開始し、「タイマーを30分に設定したよ！」と応答する

#### Scenario: 複合時間でタイマー設定
- **WHEN** ユーザが `/timer 1時間30分` と入力した時
- **THEN** 1時間30分のタイマーを開始し、「タイマーを1時間30分に設定したよ！」と応答する

#### Scenario: 数値のみの入力
- **WHEN** ユーザが `/timer 30` と入力した時
- **THEN** 30分として解釈し、タイマーを開始する

#### Scenario: 空白入りの時間指定
- **WHEN** ユーザが `/timer 1時間 30分` と入力した時
- **THEN** 空白を無視して1時間30分として解釈し、タイマーを開始する

#### Scenario: ゼロ値の指定
- **WHEN** ユーザが `/timer 0分` と入力した時
- **THEN** 「0より大きい時間を指定してください」というエラーメッセージを返す

### Requirement: タイマーの通知
システムはタイマー完了時に吹き出し（thought bubble）で通知しなければならない（SHALL）。

#### Scenario: タイマー完了通知
- **WHEN** 設定された時間が経過した時
- **THEN** `shiritagari-thought` イベントを `{ inference: "⏰ タイマーが完了したよ！", confidence: 1.0 }` 形式で emit し、吹き出しに表示する

### Requirement: 時間パースのエラーハンドリング
システムは不正な時間指定に対してエラーメッセージを返さなければならない（SHALL）。

#### Scenario: 引数なし
- **WHEN** ユーザが `/timer` とだけ入力した時
- **THEN** 使い方を案内するメッセージを返す

#### Scenario: パース不能な引数
- **WHEN** ユーザが `/timer abc` のように数値を含まない引数を入力した時
- **THEN** 「時間を解析できませんでした」というエラーメッセージと使い方を返す
