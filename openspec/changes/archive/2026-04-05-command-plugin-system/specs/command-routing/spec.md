## ADDED Requirements

### Requirement: スラッシュコマンドの解析
システムはユーザ入力が `/` で始まる場合、コマンド名と引数に分割して解析しなければならない（SHALL）。

#### Scenario: コマンドの解析
- **WHEN** ユーザが `/timer 3時間` と入力した時
- **THEN** コマンド名 `timer`、引数 `3時間` として解析する

#### Scenario: 引数なしコマンド
- **WHEN** ユーザが `/help` と入力した時
- **THEN** コマンド名 `help`、引数は空文字列として解析する

### Requirement: コマンドのディスパッチ
システムは解析されたコマンド名に一致する登録済みプラグインを検索し、実行しなければならない（SHALL）。一致するプラグインがない場合はエラーメッセージを返す。

#### Scenario: 登録済みコマンドの実行
- **WHEN** コマンド名に一致するプラグインが登録されている時
- **THEN** そのプラグインの `execute` を引数付きで呼び出し、結果を返す

#### Scenario: 未登録コマンドの処理
- **WHEN** コマンド名に一致するプラグインが存在しない時
- **THEN** 「不明なコマンドです。/help で利用可能なコマンドを確認してください」というエラーメッセージを返す

### Requirement: コマンドとチャットの分離
システムは `/` で始まる入力を常にコマンドとして処理し、`/` で始まらない入力を従来のチャット処理で実行しなければならない（SHALL）。`/` で始まるメッセージが未知のコマンドであっても LLM チャットにフォールバックしてはならない。

#### Scenario: 通常メッセージのチャット処理
- **WHEN** ユーザが `/` で始まらないメッセージを送信した時
- **THEN** 従来の LLM チャット処理を実行する

#### Scenario: スラッシュ単体の入力
- **WHEN** ユーザが `/` とだけ入力した時
- **THEN** 「不明なコマンドです。/help で利用可能なコマンドを確認してください」というエラーメッセージを返す

### Requirement: プラグインの登録
システムは `CommandPlugin` trait を実装した任意のプラグインを `CommandRouter` に登録できなければならない（SHALL）。同名のコマンドが既に登録されている場合は警告を出す。

#### Scenario: プラグインの登録
- **WHEN** 新しいプラグインを `CommandRouter` に登録した時
- **THEN** そのプラグインの `name()` でコマンドが利用可能になる

#### Scenario: 重複登録の検出
- **WHEN** 既に登録済みのコマンド名と同名のプラグインを登録しようとした時
- **THEN** 警告ログを出力し、後から登録されたプラグインで上書きする

### Requirement: ヘルプコマンド
システムは `/help` コマンドで登録済みの全コマンド一覧とその説明を表示しなければならない（SHALL）。

#### Scenario: ヘルプの表示
- **WHEN** ユーザが `/help` と入力した時
- **THEN** 登録済みコマンドの名前・説明・使い方を一覧で返す

### Requirement: コマンドエラーのUI表示
システムはコマンド実行時のエラーメッセージをユーザに表示しなければならない（SHALL）。フロントエンドは `send_message` の失敗時にエラー内容を thought bubble に表示する。

#### Scenario: コマンドエラーの表示
- **WHEN** コマンド実行が `Err` を返した時
- **THEN** エラーメッセージを thought bubble に表示する
