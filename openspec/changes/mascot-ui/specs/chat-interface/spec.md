## MODIFIED Requirements

### Requirement: 常駐チャットウィンドウ
システムはシステムトレイに常駐し、デスクトップマスコットウィンドウを表示/非表示できなければならない（SHALL）。ウィンドウは透過背景・タイトルバーなしで、キャラクター画像・吹き出し・入力欄を表示する。

#### Scenario: システムトレイからの表示
- **WHEN** ユーザがシステムトレイアイコンをクリックした時
- **THEN** マスコットウィンドウを表示する

#### Scenario: チャットウィンドウの非表示
- **WHEN** ユーザがウィンドウを閉じた時
- **THEN** アプリを終了せず、ウィンドウを非表示にしてシステムトレイに格納する

### Requirement: AIからの質問表示
システムは行動推論エンジンからの質問を発話吹き出しとして表示しなければならない（SHALL）。

#### Scenario: 質問の通知
- **WHEN** 行動推論エンジンが質問を生成した時
- **THEN** 発話吹き出しに質問を表示し、ウィンドウを最前面に浮上させる

#### Scenario: 質問への回答
- **WHEN** ユーザがキャラクター下部の入力欄に回答を入力して送信した時
- **THEN** 回答を行動推論エンジンに渡し、エピソード記憶として保存する

### Requirement: ユーザからの能動的な対話
システムはキャラクター下部の常設入力欄からユーザの自由なメッセージを受け付け、記憶を活用して応答しなければならない（SHALL）。

#### Scenario: 通常のチャット
- **WHEN** ユーザが自発的にメッセージを送信した時
- **THEN** 記憶ストア（パターン、エピソード、推測ログ、プロファイル）をコンテキストとしてLLMに渡し、応答を発話吹き出しに表示する

#### Scenario: 過去の操作についての質問
- **WHEN** ユーザが「さっき何してたっけ」のような質問をした時
- **THEN** 推測ログとエピソード記憶を参照し、該当する情報を含む応答を返す

## REMOVED Requirements

### Requirement: チャット履歴表示
**Reason**: マスコットUIではチャット履歴のスクロール表示を廃止し、吹き出しによる最新メッセージのみの表示に切り替えるため
**Migration**: 過去の対話内容は引き続きDBに保存される。UIでは吹き出しに最新の1メッセージのみ表示する
